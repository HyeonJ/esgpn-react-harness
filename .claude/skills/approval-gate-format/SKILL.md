# Approval Gate Format

하네스가 사용자 판단을 기다릴 때 반드시 다음 포맷으로 출력한다. 사용자가 "무엇을 봐야 하고 무엇을 답해야 하는지"를 혼동 없이 알 수 있도록 한다.

## 표준 포맷

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⏸ 승인 대기 중

📍 현재 단계: {Phase/Step 이름 — 예: "Phase 3 / 섹션 Hero / 단계 2 (계획)"}
📄 검토할 파일: {상대경로 — 예: "plan/main-hero.md"}
🎯 결과 요약:
   - {3~5줄 bullet — 주요 산출물, 수치, 결정 사항}

👉 다음에 당신이 할 수 있는 행동:
   [A] "{짧은 명령 예시}" — {그 결과}
   [B] "{짧은 명령 예시}" — {그 결과}
   [C] "{짧은 명령 예시}" — {그 결과}
   [D] "{짧은 명령 예시}" — {그 결과}

💡 권장: {한 문장으로 "이런 경우 A, 저런 경우 B" 가이드}

✅ 검토 체크리스트:
   - [ ] {이 단계 특유의 확인 항목 3~6개}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 멈춤 지점별 템플릿

### 1. Phase 1 plan 작성 완료
- 선택지: [A]승인·구현 시작 [B]메모 반영 [C]스택 변경 [D]취소
- 체크: npm 패키지 목록, 스크립트 4개, 토큰 추출 전략, 폰트 보정 전략

### 2. Phase 2 페이지 분할 완료
- 선택지: [A]승인·첫 섹션 진행 [B]섹션 재분할 [C]섹션 합치기/쪼개기 [D]리서치 보완
- 체크: 섹션 수, 각 섹션 12K 이하, 사전 추정과 차이, 신규 공통 컴포넌트, 베이스라인 스크린샷 확보

### 3. 섹션 리서치 완료
- 선택지: [A]plan 작성 진행 [B]리서치 보완 [C]에셋 목록 재확인 [D]캔버스-에셋 불일치 건 결정
- 체크: 에셋 개수 일치, 동적 여부 명시, 공통 컴포넌트 재사용 식별, Figma 베이스라인 존재

### 4. 섹션 plan 작성 완료
- 선택지: [A]승인·구현 시작 [B]메모 반영 [C]컴포넌트 구조 변경 [D]새 패키지 승인 필요
- 체크: 컴포넌트 200줄 이하 예상, 에셋 1:1 매핑, 동적 에셋 정적 추출 명시, 반응형 전략, 새 패키지 여부

### 5. 8 게이트 측정 완료 (통과) — 보통 자동 커밋 후 통보만
현재 기본 동작: 모든 게이트 PASS + 육안 OK → 오케스트레이터가 자동 커밋 (`[auto]` 태그) + PROGRESS.md 갱신 후 다음 섹션 진행. 사용자 개입은 **명시적 요청 시에만**.

- 선택지: [A]자동 진행 유지 [B]diff 이미지 재확인 후 진행 [C]재측정 요청
- 체크: G1~G4 수치, G5~G8 수치, 육안 semantic 확인, 빌드/타입 통과

### 6. 게이트 3회 실패 (완화는 맨 끝)
- 선택지 (순서 중요):
  - [A] **섹션 재분할** — 이 섹션이 너무 큼. 이질적 에셋 3+/반복 자식 3+ 있으면 쪼개기 먼저
  - [B] **다른 접근** — 재구성 전략 변경 (raster → HTML, blend 모드 교체 등)
  - [C] **엔진 차이 수용** — hard-light blend, 한글+blend 6~7% 수렴 패턴 등 구조적 불가피 확인 → `docs/tech-debt.md`에 `ACCEPTED` 등록
  - [D] **되돌리기** — 섹션 재시작
  - [E] **완화 (최후)** — G1/G4 등 픽셀/색상만 적용 가능. G5~G8 품질 게이트는 완화 불가. 선택 시:
    - 커밋 메시지 `[ACCEPTED_DEBT]` 태그 필수
    - `docs/tech-debt.md`에 `OPEN` 엔트리 자동 추가 (파일·증상·현 수치·리팩터 방향)
    - `OPEN` 부채 3건 이상이면 다음 섹션 진행 차단됨
- 체크: 3회 시도 내역, 현재 diff %, 의심 원인, diff 이미지 확인 여부, 재분할 가능성 검토했는지
- **금지**: [E] 완화를 첫 옵션처럼 제시하는 것 — 재분할/다른접근/엔진차이 확인이 선행되어야 함

### 7. 페이지 통합 검증 완료
- 선택지: [A]페이지 완료 처리·다음 페이지 [B]반응형 재검증 [C]크로스 브라우저 재검증 [D]이슈 수정
- 체크: 전체 diff < 8%, 3 뷰포트, 3 브라우저, 섹션 경계 정상

## 작성 규칙

- **결과 요약은 숫자 중심** — "잘 됐습니다" 금지, "diff 3.2%, 에셋 5/5, 빌드 성공"
- **선택지 명령어는 짧게** — 사용자가 그대로 복사해 쓸 수 있어야 함
- **권장 문장은 조건부** — "에셋 불일치 없으면 A, 의심되면 D"
- **체크리스트는 해당 단계에 특화된 것만** — 일반 체크는 뺀다
- 이 포맷을 **생략·축약 금지**. 멈춤 지점에선 항상 전체 포맷 출력.
