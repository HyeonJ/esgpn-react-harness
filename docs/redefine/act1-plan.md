# 1막 — ESGPN 완주 + 구조 품질 재측정

> 기간 목표: 2~3주
> 역할: 현재 자산의 **진짜 가치**를 구조 기준으로 측정. 기존 섹션은 **절대 리팩터하지 않는다.** 측정만 한다.
> 출구 시점: 9페이지 머지 + 43섹션 구조 지표 확보 + 회고 답변 완료

---

## 0. 이 막의 원칙

1. **회귀 금지.** 기존 섹션을 새 기준으로 재구현하지 않는다. 지금 바꾸면 측정 대상 자체가 훼손된다.
2. **측정만 한다.** 나쁜 결과가 나와도 1막에서는 기록만 하고 2막으로 넘긴다.
3. **체감 대신 숫자.** "대충 괜찮아 보임" 금지. 모든 판단에 지표를 붙인다.
4. **회고는 완주 직후.** 시간이 흐르면 판단이 흐려진다. 완주 당일~익일에 답변 작성.

## 2. 회고 질문 (완주 직후 답변)

### Q1. 픽셀 <5%인데 구조가 망가진 섹션은?
- 후보: `main-hero` 카드, `contest-benefits` CTA (이미 tech-debt 식별됨)
- 1막에서 추가로 발견된 건?
- 각 섹션마다 "구조 망가진 정도"를 지표로:

| 섹션 | G1 | text-raster | 마법숫자 | 토큰참조율 | 판정 |
|---|---|---|---|---|---|
| main-hero | 2.24% | ... | ... | ... | ... |
| ... | | | | | |

### Q2. G1 <5% PASS가 실제 유지보수성에 얼마나 기여했나?
- 섹션 완료 후 수정이 필요했던 케이스 나열
  - 예: 반응형 이터레이션에서 `main-hero` 재작업 범위가 얼마나 컸나?
- 수정 비용이 컸던 섹션의 공통점은? (raster 포함? absolute 다수? 마법 숫자?)
- 수치로: "G1 < 5% 섹션의 평균 재작업 파일 수 vs ≥5% 섹션의 평균"

### Q3. 만약 G1 기준이 15%였다면 섹션당 공수가 얼마나 줄었을까?
- 3회차 이상 수정 반복한 섹션 카운트
- 그 섹션들이 15% 기준이었다면 몇 회차에 끝났을지 추정
- 실제 절감 공수 시간 추정 (섹션당 평균 h × 절감 회차)
