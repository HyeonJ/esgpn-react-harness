# Dynamic Asset Handling

**Figma 캔버스의 정적 표현이 진실의 원천이다.** Figma MCP가 원본 GIF/비디오를 주더라도 디자이너가 캔버스에서 본 것은 정지 프레임이다. 움직이는 원본을 그대로 사용하면 검정 배경 + 의도되지 않은 움직임 + 합성 오류가 발생한다.

## 동적 에셋 식별 (3가지 — 하나라도 해당하면 동적)

1. **확장자 검사**: MCP 응답 URL 확장자가 `.gif`, `.mp4`, `.webm`, `.mov`, `.apng`
2. **파일 타입 검사**: 다운로드 후 `file <파일>` 결과가 `GIF image data`, `ISO Media`, `WebM`, `QuickTime`
3. **노드 타입 검사**: `get_design_context` 응답 노드 타입이 `VIDEO` 또는 미디어 컨테이너

## 캔버스-에셋 일치 검증 (필수)

research의 에셋 목록 작성 후 반드시 수행:
- 캔버스에서 보이는 시각적 요소 개수 ↔ 추출된 에셋 목록 항목 개수
- 개수 불일치 또는 정적 영역에 동적 에셋 매핑 시 → **사용자에게 보고하고 멈춘다**
- 예: 캔버스에 정적 일러스트 1개 → MCP가 video 노드 + 정적 이미지 2개 반환 → 케이스 분석 필요

## 처리 절차

### 정적 에셋 (PNG, JPG, SVG)
```
1. scripts/download-assets.sh 로 src/assets/{섹션명}/raw/ 다운로드
2. scripts/verify-assets.sh 로 file 타입 확인, 불일치 시 rename
3. scripts/process-assets.py 로 후처리 (검정 배경 → 투명, SVGO 등)
4. 최종 파일은 src/assets/{섹션명}/ 루트
```

### 동적 에셋 (GIF, MP4, WebM 등)
```
1. 원본 다운로드 금지 (또는 받아도 코드에서 사용 금지)
2. 해당 노드의 부모 컨테이너 노드 ID로 figma-rest-image.sh 호출
   - 옵션: PNG, 해상도 2x+, 투명 배경 (가능하면)
   - 부모 컨테이너 단위: 합성 결과(오버레이/마스크 포함)가 그대로 들어옴
3. src/assets/{섹션명}/{이름}-static.png 로 저장
4. plan 에셋 카탈로그 "처리 방식": "정적 프레임 추출 (figma-rest-image.sh, 부모 노드 {ID})"
5. 추출 PNG를 눈으로 확인하여 Figma 캔버스 표현과 일치하는지 검증
```

## 금지 사항

- ❌ GIF를 `<img>` 태그로 그대로 사용
- ❌ GIF를 APNG로 변환 (여전히 움직임 유지)
- ❌ 비디오를 `<video>` 임베드 (Figma 캔버스에 비디오 없으면 의도된 게 아님)
- ❌ video 노드 + 정적 오버레이를 코드에서 합성 시도 → **반드시 부모 컨테이너 단일 캡처로 대체**

## 의심스러우면 (폴백)

모호한 경우 **그 영역의 부모 컨테이너를 통째로 `figma-rest-image.sh`** 으로 떠서 단일 PNG로 사용한다. 개별 에셋 합성 시도보다 단일 캡처가 항상 더 정확하다.

## 예외

사용자가 명시적으로 "이 부분은 움직이게 해줘"라고 지시한 경우에만 동적 에셋을 원본 그대로 사용. 예외 적용 시 plan/{섹션명}.md에 사용자 지시 내용을 명시적으로 기록.

## 에셋 목록 표 포맷 (research에 사용)

```markdown
| # | 종류 | Figma URL | 노드 ID | 부모 사이즈 | 원본 사이즈 | 동적 여부 | 처리 방식 |
|---|------|-----------|---------|-------------|-------------|-----------|-----------|
| 1 | SVG  | https://... | 12:345 | 28x28 | 21x8.75 | 정적 | 부모 div + flex center |
| 2 | GIF  | https://... | 12:347 | 400x400 | 400x400 | **동적** | **정적 프레임 추출 (figma-rest-image.sh, 부모 12:346)** |
```
