## 신규 공통 컴포넌트 후보

기존 카탈로그(`docs/figma-project-context.md §5`): Header, Footer, Top Nav.

본 페이지에서 식별:

1. **StatItem** — Stats 섹션 (4개) 및 향후 다른 페이지에서 재사용 가능. 큰 숫자 + 작은 캡션 2줄 구조 (`29:381`/`29:380`/`29:379`/`29:378`).
2. **ProgramCard** — Programs 3개 카드가 동일 골격 (그린 BG + 진행바 + 타이틀 + 체크리스트 + 본문 + 자세히 보기 CTA). 좌우 부유 장식만 다름. 페이지 내부 컴포넌트로 추출 권장 (현 단계에선 공통 X, 페이지 로컬 컴포넌트).
3. **NewsCard** — News 섹션의 5개 카드 (`43:315` 내부, 메타 비어있어 단계 1에서 확인 필요). 다른 페이지에서 사용될 가능성 높음.
4. **Section CTA Button (그린 pill)** — Hero, Gallery 등에서 반복되는 "ESG 실천 아이디어 경진대회 참여하기", "ESG-마인드 자격검정 신청하기", "전체보기" 등. **공통 Button 컴포넌트** 도입 권장.

## 리스크 메모

`docs/figma-project-context.md §7` 페이지 공통 리스크 외:

1. **메타 비어있는 노드 3개 (`43:1730` Hero, `43:315` News, `43:545` Gallery)** — 자식 노드가 metadata에 노출되지 않음. 단계 1(섹션 리서치)에서 `get_design_context`로 실제 구조·텍스트·이미지 fill 추출 필수. 단일 평탄 이미지로 export된 디자인일 가능성 있음 → 그렇다면 이미지 슬라이싱이 불가능해 텍스트 재구성 필요.
2. **Intro 섹션 globe 이미지 (`26:92` `image 17` 419×404)** — 회전 애니메이션 GIF 가능성. 단계 2에서 fill 검사 후 동적 에셋이면 `dynamic-asset-handling` 스킬로 정적 PNG 추출. (사전 리스크 §7과 동일)
3. **Programs Card 부유 장식 (city 타일, leaf)** — 카드와 분리된 floating decoration. 절대 위치 + 음수 z-index 또는 SVG mask 처리 필요. 단계 2 plan에서 좌표·크기 정확히 측정해야 G2 통과 가능.
4. **Stats 섹션 connector (`29:324` 등 6개)** — 동적 SVG 라인. 픽셀 정확도 어려움. SVG로 export하거나 inline path 작성.
5. **사전 추정 vs 실제 mismatch** — Partners/CTA 섹션이 존재하지 않으므로 README/PROGRESS의 페이지 진척률 계산은 9개 기준으로 재산정 필요.
6. **Footer offset 충돌** — Footer instance(y=9378)와 페이지 wrapper 자식 frame(`43:486`, height=10086)이 9px 오버랩. 페이지 height 10077과 wrapper height 10086 차이 (+9px) 정보 정합성 확인 필요. (구현 시 Footer는 RootLayout이 처리하므로 영향 미미하지만 baseline diff 측정 시 주의)
