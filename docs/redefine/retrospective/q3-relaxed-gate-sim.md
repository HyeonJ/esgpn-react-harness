# Q3 — G1 기준이 15%였다면 섹션당 공수가 얼마나 줄었을까?

> 측정일: 2026-04-16
> 데이터 소스: `PROGRESS.md` (회차 기록) + `git log` (commit 메시지의 G1 추이) + `docs/tech-debt.md`
> 작성 원칙: 시뮬레이션은 추정치임을 명확히. 회차 시간은 평균 1~2h 추정으로 폭 제시.

---

## 0. 핵심 결론 (TL;DR)

- **3회차 이상 진행 섹션 = 10/39 (26%).** 누적 회차 합계 약 **34회차 추가**.
- 만약 G1 게이트가 15%였다면 → **추정 ~25회차 절감** (10 섹션 평균 2.5회차씩).
- 회차당 공수 1~2h 가정 시 → **25~50h 절감 (≈ 3~6 working day, 약 1 working week)**.
- 그러나 "절감"이 곧 "이득"은 아님. **잃는 것**: G1 < 5% PASS 압박이 약해지면 G1 chase에서는 자유롭지만, 오히려 absolute_count·magic_number가 더 늘어났을 수도 있음 (압박 부재 → 정밀도↓). 측정 불가, **반사실(counterfactual) 한계**.
- **순이득 추정**: 약 +20~35h (절감 시간의 70%) — main-hero 같은 G1 chase 케이스만 막아도 명백한 이득. 나머지는 toss-up.

---

## 1. 측정 방법

1. PROGRESS.md에서 각 섹션의 회차 기록 추출 ("3회차 통과", "1회차 완통과" 등).
2. git log commit 메시지에서 회차별 G1 수치 추적 (예: `8.07% → 5.47%`).
3. 가상 게이트 = 15% 적용 시 각 섹션의 round-1 G1이 15% 미만이었다면 그 시점 종료로 간주.
4. round-1 G1을 알 수 없는 경우 final G1 + 회차당 ~2~3%p 개선 가정으로 역산.
5. 회차당 공수: research/impl/measure/iter 사이클 1~2h 추정.

**한계 (Caveat)**:
- Round-1 G1 수치가 모든 섹션에 명시되어 있지 않음. 추정 구간으로 폭 제시.
- 회차당 공수 1~2h는 자율 모드(experiment/autonomous-run)와 수동 모드 평균. 케이스별 편차 큼.
- 게이트가 바뀌면 사람의 행동도 바뀜 — 절감 시간이 다른 곳에 쓰였을 수도 있고, 게이트가 약해진 만큼 다른 안티패턴(absolute, magic) 증가 가능. **반사실은 측정 불가.**

### 특이 케이스

- **main-hero**: round-1 G1이 이미 4.49%로 5% 미만이었음. 5회차 추가는 **G1 게이트 통과 후에도 "더 잘 맞춰보자"는 자체적 공수 투입.** 즉 게이트 5%였어도 1회차 종료 가능했고, 15% 게이트로 바꿔도 똑같이 1회차 종료. **주요 절감 원인은 G1 chase 문화.**
- **certification-hero**: 최종 10.98%인데 ⚠ ACCEPTED. 15% 게이트라면 round 1~2에서 정상 PASS — 완화 토의·tech-debt 등록 절차 자체 불필요.
- **contest-benefits 4회차**: 회차 2에서 multiply 실험 → G1 악화 (10.93~12.21%) → revert. **15% 게이트라면 round 1(9.48%)에서 종료. 실험 자체가 발생 안 함.**

## 4. 잃는 것 — 반사실(counterfactual) 분석

15% 게이트로 바꾸면 절감 시간만큼 다른 곳에 비용이 발생할 수 있다.

### 4.1 명백히 이득인 부분 (추정 70%)

- **main-hero G1 chase**: 5 회차 + 4 commit revert. icon 배경 제거 4회 시도(chroma-key, luma-to-alpha, rembg)는 모두 G1 5%대로 끌어내리려는 시도. 15% 게이트라면 1회차 종료 후 **다른 섹션으로 즉시 이동**. 명백한 순이득 ~10h.
- **certification-hero, contest-benefits, gallery-agreements 완화 절차**: 3건 × 0.5h = 1.5h 절감 (회의·문서화).
- **revert/실험 commits**: 약 5개 commit (main-hero 4 + contest-benefits 1) ≈ 8h.

→ **명백한 순이득: 약 20h.**

### 4.2 불확실한 부분 (toss-up, 추정 30%)

- 게이트 압박이 약해지면 absolute coordinate 사용·magic number 박제가 더 늘었을 가능성. 1막 평균(magic 14.6/섹션, abs 평균)이 15% 게이트 하에서는 더 높았을 수도. **측정 불가.**
- "잘 맞춰보자"는 자체 욕구는 게이트 무관하게 발생할 수도 있음 (main-hero 사례). 실제로 게이트 5%여도 4.49% 시점에서 PASS 했는데 5회차 더 진행함.
- 절감 시간이 다른 가치 작업으로 이동했다는 보장 없음. 게이트 약하면 하루 더 쉬거나 다른 비효율로 흡수될 수 있음.
- **수동 시각 회귀 비용 전가**: G1 게이트가 헐거워지면 자동 PASS되는 섹션이 늘어남 → 디자이너/QA가 눈으로 보고 "여기는 너무 깨졌으니 고쳐달라" 지시하는 **사람-사람 커뮤니케이션 핑퐁**이 늘어날 수 있음. AI 작업 시간은 줄어도 사람 의사결정 시간이 늘면 총 leadtime은 비슷할 수 있음. 측정 불가.

→ **toss-up: 절감 시간의 30%는 다른 비용으로 상쇄될 수 있음.**

### 4.3 잃을 수 있는 것 — 정성적

- **G1 < 5% 압박이 만든 "정밀도 문화"**: round-1 4.49%에서 만족하지 않고 2.24%까지 끌어내린 main-hero는 픽셀 정밀도 학습 자료가 됨 (rembg/luma 스크립트 자산화). 15% 게이트면 이 자산은 만들어지지 않음.
- **완화 결정의 학습 효과**: ⚠ ACCEPTED 사례들이 docs/section-implementation.md §2.5 (한글+blend 6~7% 수렴 패턴) 같은 노하우로 축적됨. 게이트 약하면 이 노하우 부재.

## 5. 종합 — 순이득 추정

| 항목 | 추정 |
|---|---|
| 명백한 절감 (G1 chase + revert + 완화 절차) | **~20h** |
| 추가 절감 가능성 (round-2~3 단축) | **~15h** |
| 반사실 비용 (다른 안티패턴 증가, 정밀도 문화 손실) | **-10h** (정성적 손실 별도) |
| **순이득 (시간만)** | **~25h** |

**판정**: 시간 측면에서는 명백한 이득. 정성적으로는 **"정밀도 노하우 자산화"라는 부수 효과를 포기**해야 함. 2막 게이트 설계 시 트레이드오프 명시 필요.

## 6. 2막 시사점

1. **G1 게이트 = 15% (참고 지표) + 구조 게이트 = 차단 (philosophy.md §4 위계)** 이 정량적으로 타당.
   - 절감 시간(~25h)을 5섹션 만성염증 리팩터에 투입하면 1~2 섹션 완전 리팩터 가능.
2. **자율 모드(experiment/autonomous-run)에서 G1 게이트 5% 강제는 비용 ↑↑.** 사람 개입 비용 외에 LLM 토큰 비용도 회차당 누적. 자율 모드는 **G1 ≤ 15% + 구조 강제** 조합이 ROI 높음.
3. **G1 chase 자체에 대한 stop-loss 규칙 필요**: round-1 G1이 이미 게이트 통과인데 추가 회차 진행하는 경우(main-hero) 명시적 stop. "Good enough" 정의 필요.
4. **정밀도 학습 자산은 별건 R&D 트랙으로 분리**: main-hero icon 배경 제거 같은 실험은 섹션 구현이 아닌 도구 개발 트랙(scripts/rembg-icon.py 자산화)에서 진행. 섹션 작업 회차에 묶지 않음.
